# План реализации — ветка `alisa` (максимальный скор на SelTox)

Единый синтез: наша реализация (`selt-mvp`), исходный `main`-план, `merge_plan_alisa.md` и
`CHEMX_FINDINGS.md`. Идеи конспекта и наш код взвешены равнозначно: где наше лучше — оставляем
наше; где конспект прав — берём; где конспект ошибается — отбрасываем с доказательством.

---

## 1. Модель очков: где реально лежат баллы

Рубрика: **40** за победу над single-agent baseline на ≥1 домене + **20** UI + **20** код/README +
**20** презентация = 100. Бонусы: **+5**/домен (до +40), **+10** за оба направления (mol + nano).

Из `progress.md`: **blank-prior уже даёт 0.073 > baseline 0.045** — 40 очков почти взяты ещё до LLM,
*если* наш evaluator совпадает с официальным. Игра распадается на: (а) сделать победу железной
(Port A), (б) раздуть отрыв настоящей экстракцией, (в) бонусы по остаточному времени.

### Где очки внутри SelTox (по `metrics_seltox_from_single_agent.csv`)

- **6 полей с baseline F1 = 0.0**: `np_synthesis, concentration, time_set_hours,
  temperature_for_extract_C, duration_preparing_extract_min, ph_during_synthesis`. Каждое = 1/23 ≈
  **0.043 Macro-F1** при доведении до 1.0. Это параметры green-synthesis из текста methods
  («70 °C, pH 10, 2 h, AgNO₃») — легко извлекаются, baseline их галлюцинирует → чистый апсайд.
- Даже «топовые» поля низкие (`np` 0.125, `bacteria` 0.123, `method` 0.115) → потолок везде высокий.
- Почему blank бьёт baseline: разреженные поля в GT = `nan`/`NOT_DETECTED`; baseline галлюцинирует
  (FP), мы абстейним (TP). **Главный рычаг — дисциплина воздержания, а не точность чисел.**

### Тонкость метрики: гранулярность строк

Метрика — мультимножество внутри статьи, wrapper делает `drop_duplicates()`. Значит:

- одинаковые пустые строки схлопываются в одну → одна blank-строка недобирает recall на many-rows
  статьях;
- **оптимум — одна строка на реальный эксперимент (bacteria × np)** с `nan` в неизвестном: строки
  различаются по `bacteria`, не дедуплицируются, дают нужное число копий `nan` в разреженных
  колонках → recall растёт «бесплатно».

**Энумерация строк сама по себе — рычаг скора**, а не только извлечение полей.

## 2. Что берём / что отбрасываем (идеи равнозначны)

**Оставляем наше (лучше конспекта):**

- table-faithful ensemble-парсер (PyMuPDF + pdfplumber + camelot) — лучше marker-shred у baseline;
- детерминированные `normalize.py` + `ValidatorAgent` (в конспекте только лозунги);
- evaluator с HF→local cache + Port A-строгость; находка blank-prior; Yandex-клиент.

**Берём из конспекта:**

- **U1** длинный контекст (deepseek 1M) → подаём весь документ, а не BM25 top-k;
- **U2** инъекция официальной `seltox.json` + few-shot из реального gold;
- **U3** калибровка нормализаторов под фактический формат GT;
- **U4** селективный VLM по страницам-таблицам;
- акцент на abstention; список «чего НЕ делать».

**Отбрасываем/исправляем в конспекте (с доказательством):**

- ❌ «дешёвая победа на метаполях» — `metric_calc.py:193` скорит только `EXTRACTED_COLUMNS`; эталоны
  `cocrystals` = 7 строк, `seltox` = 23, **никаких** `page_*/doi/journal`. CrossRef/OpenAlex — только
  для скачивания PDF, не ради метрики.
- ❌ «Co-crystals как основной домен» — премиса (метаполя) ложна; co-crystals 0.296, 70 строк,
  `SMILES F1=0`, нужен OCSR. Поправка усиливает выбор **SelTox**.

**Упрощаем у себя (Simplicity First + «mono-agent конкурентоспособен»):**

- BM25 evidence-finder → понижаем до тегера provenance для UI (не бутылочное горло);
- 3 LLM-агента (activity/np/synthesis) → один schema-driven вызов по всему документу;
- никакого self-consistency / ensembling / fine-tuning.

## 3. Целевая архитектура (один проход, table-faithful, abstention-first)

```
PDF (+ supplementary)
 → 1. Parse (наш ensemble): text + таблицы→markdown, page→PNG, кэш
 → 2. Context (U1): весь текст + таблицы как markdown с тегами page/section
 → 3. Extract (U2, один вызов): deepseek-v4-flash, response_format=json_schema(seltox.json, strict),
       temperature 0, few-shot из gold (вне eval-сета); одна строка на (bacteria×np); абстейн где не сказано
 → 4. Normalize (U3, калибровано под GT): числа/бактерии/np/units/mdr(0/1)/missing + sanity-диапазоны
 → 5. Validate: убрать неподтверждённое/галлюцинации, типы по схеме, абстейн на разреженных без сильного evidence
 → 6. Enumerate/dedup: гранулярность (bacteria×np), drop_duplicates-aware
 → 7. Submission: blank-prior на каждый target PDF + извлечённые строки; имя pdf в формате метрики (регистр!)
 → 8. (опц.) VLM по страницам, где парсер таблиц упал
 → 9. Evaluate (Port A, доверенный): ours vs baseline по полям + Macro-F1
```

## 4. План реализации по фазам (с go/no-go)

### Gate A (СНАЧАЛА, у тебя в среде) — доверенный evaluator + разрешение бага регистра

`validate-evaluator` уже написан. Запуск решает критический вопрос: `SELTOX_ARTICLES` — mixed-case,
а baseline лучит gold/pred, но итерирует набор «как есть». Два исхода:

- режим **`articles-as-is`** совпал с эталоном → официальная метрика матчит только статьи с именами в
  нижнем регистре (~17/65), **потолок Macro-F1 ≈ 0.26**, наш «здравый» evaluator (все 65) завышает
  скор → выравниваем оценку и формат имён сабмишена под официальное поведение;
- режим **`lowercase-articles`** совпал → бага нет, считаем по всем 65.

**Пока Gate A не пройден — ни одно число не доверенное.** Всё ниже опирается на его результат.

### Фаза 1 — Калибровка под GT (Port G) [нужен gold]

Команда-инспектор формата: по каждому из 23 полей dtype/%nan/уникальные/точность чисел/написание
бактерий/строки единиц. Записать в `docs/gt_format.md`. Перенастроить `normalize.py` точно под GT
(числа, `mdr`=0/1, токены пустоты, словарь `method`/`shape`, формат `np`/`bacteria`).
**Verify:** normalized-GT-as-pred остаётся ≈ 1.0.

### Фаза 2 — Сабмишен-форма + пол абстиненции (формализуем blank-prior)

`build-submission`: blank-prior (правильные `nan`/`NOT_DETECTED`, имена pdf по итогу Gate A) + место
под извлечённые строки. **Verify:** под доверенным evaluator ≥ floor.
**Это запирает 40 очков почти без риска ещё до LLM.**

### Фаза 3 — Один schema-driven экстрактор (Ports F+E, схлопывание)

`complete_json(seltox.json, strict)` + few-shot из gold вне eval; full-doc контекст; энумерация
(bacteria×np); + normalize (Ф1) + validate. **Verify:** на ag_nps-статье `pH 10, 70 °C, 2 h, AgNO₃,
8–41 nm` извлеклись в формате GT. **Gate:** prior+extraction > prior-only под доверенным evaluator.

### Фаза 4 — Покрытие PDF (Port B) [нужна сеть]

Unpaywall/OpenAlex/publisher + **supplementary** (MIC/ZOI-таблицы часто в SI). Поднять с 6/65,
честно фиксировать N/65.

### Фаза 5 — Итерации до максимума SelTox

Цикл по полям в порядке (headroom × частота): сперва 6 нулевых (green-synthesis из methods), затем
строки `np/bacteria/method/shape/precursor` (формат), затем `mic/zoi/sizes` (числа+таблицы).
Инструменты: правила промпта, few-shot, нормализация, точечный VLM.
**Gate:** Macro-F1 уверенно > 0.045 (цель ≥ 0.12–0.20 или близко к потолку при баге).

### Фаза 6 — Селективный VLM (Port H, опц.)

base64-адаптер в `LLMClient` + qwen3.6; маршрутизировать только упавшие/низко-уверенные
страницы-таблицы. **Verify:** растут именно `mic/zoi/sizes` без перерасхода бюджета.

### Фаза 7 — UI + README + презентация (другие 60 очков)

Streamlit: upload→parse→extract→таблица + provenance (page/table) + скачать CSV + плашка
ours-vs-baseline. README: one-command reproduce (Makefile), `validate-evaluator` как «доказательство
доверия к метрике», пиннинг. Презентация: владение метрикой (баг регистра + поправка про метаполя),
архитектура, per-field таблица.

### Фаза 8 — Бонусы (по EV)

**Synergy (+5)** — ближайший nano, переиспользует весь стек (схема + few-shot + дельта нормализатора)
→ дёшево. **Complexes (+15: +5 домен +10 оба направления)** — только если SelTox+Synergy заперты и
есть время (планка 0.290, нужен RDKit-canonical/OCSR, риск).

## 5. Приоритет атаки по полям

1. **6 нулевых green-synthesis полей** — текст methods, чистый апсайд, низкий риск.
2. **`np`, `bacteria`, `method`** — почти в каждой статье; решает формат GT (Ф1) + few-shot.
3. **Энумерация строк (bacteria×np)** — поднимает recall разреженных полей «бесплатно».
4. **`shape`, `precursor`, `strain`** — словари/формат.
5. **`mic`, `zoi`, sizes** — таблицы (+ при упорстве VLM) и числовой формат.

## 6. Риски и митигации

| Риск | Митигация |
|---|---|
| Баг регистра / потолок | Gate A разрешает; формат имён сабмишена и таргет-цифры под официальное поведение |
| Над/недо-энумерация строк | гранулярность (bacteria×np) + validator + dedup-aware |
| Нормализатор расходится с GT | Ф1 с GT-as-pred sanity |
| Галлюцинации на разреженных | abstention-first промпт + validator + blank-prior пол |
| Хвост недоступных PDF | blank-prior всё равно скорит абстиненцией; честный отчёт покрытия |
| Офлайн dev-среда | data-шаги — в среде с сетью/gold; здесь только код+тесты+syntax |
